# Don Dog (돈독)

## Product Requirement Document (PRD)

Version 1.0

---

# 1. 프로젝트 소개

Don Dog는 대학 학생회, 동아리, 학회, 소모임 등의 회계를 AI가 자동으로 관리하고 감사해주는 AI 기반 회계 관리 플랫폼이다.

사용자는 복잡한 엑셀 장부 대신 직관적인 대시보드를 통해 회비 사용 현황을 확인할 수 있으며, AI는 거래를 자동 분류하고 회칙 위반 가능성을 분석하여 투명한 회계 운영을 지원한다.

이 프로젝트는 단순한 가계부나 관리자(Admin) 시스템이 아니다.

"신뢰를 만드는 AI 회계 플랫폼"을 만드는 것이 목표이다.

---

# 2. 프로젝트 목표

이 프로젝트는 다음 세 가지 목표를 가진다.

1.

학생회 및 동아리 회계를 누구나 쉽게 이해할 수 있도록 한다.

2.

AI가 거래를 자동 분석하여 총무의 업무를 줄인다.

3.

회계의 투명성을 높여 구성원 모두가 안심하고 사용할 수 있는 서비스를 만든다.

---

# 3. 핵심 가치

Don Dog는 아래의 가치를 가장 중요하게 생각한다.

• Transparency (투명성)

회비가 어떻게 사용되는지 모두가 쉽게 확인할 수 있어야 한다.

•

Trust (신뢰)

AI가 회칙과 거래를 검토하여 신뢰할 수 있는 회계를 만든다.

•

Simplicity (간결함)

복잡한 회계 프로그램처럼 보이지 않는다.

누구나 처음 사용해도 쉽게 사용할 수 있어야 한다.

•

Premium Experience

학생 서비스처럼 보이지 않고,
실제 SaaS 서비스를 사용하는 경험을 제공한다.

# 6. 브랜드 톤앤매너

서비스는 절대로 딱딱하면 안 된다.

그러나 너무 캐릭터 서비스처럼 가벼워서도 안 된다.

적절한 균형을 유지한다.

예시

🐶 Don Dog가 거래를 분석했어요.

🐶 문제가 없는 거래예요.

🐶 공동 승인이 필요해요.

🐶 회칙 위반 가능성이 있어요.

이처럼 AI가 사용자를 도와주는 느낌을 준다.

# 14. 개발 원칙

코드는 항상 재사용 가능해야 한다.

컴포넌트를 중복 생성하지 않는다.

디자인의 일관성을 유지한다.

Spacing을 일정하게 유지한다.

아이콘 스타일을 통일한다.

Card 디자인을 통일한다.

애니메이션을 일관되게 적용한다.

절대로
