# 목적

이 문서는 Don Dog의 시각적 방향성을 정의한다.

Cursor는 기능 구현보다

"디자인 완성도"

를 우선해야 한다.

절대로 일반적인 Admin Dashboard처럼 구현하지 않는다.

사용자는 처음 사이트를 열었을 때

학생 프로젝트가 아니라

실제 스타트업 서비스를 보는 느낌을 받아야 한다.

What Should Users Feel?

##################################################

사용자가 처음 서비스를 열었을 때

생각해야 하는 것

↓

"와 디자인 진짜 좋다."

↓

"토스 같은데?"

↓

"학생회 서비스 맞아?"

↓

"진짜 출시된 서비스 같다."

절대로

"학교 과제"

같아 보여서는 안 된다.

---

##################################################

What Should Users Never Feel?

##################################################

촌스럽다.

복잡하다.

ERP 같다.

Bootstrap 같다.

예전 관리자 페이지 같다.

엑셀 같다.

정보가 너무 많다.

버튼이 너무 많다.

색이 너무 많다.

카드가 너무 작다.

글씨가 너무 작다.

---

##################################################

Golden Rule

##################################################

디자인을 고민할 때마다

아래 질문을 한다.

이 화면이 Stripe에 있어도 어색하지 않은가?

이 화면이 토스에 있어도 이상하지 않은가?

이 화면이 Apple에서 만들었다고 믿을 수 있는가?

답이

YES

가 될 때까지 개선한다.

---

##################################################

Final Vision

##################################################

Don Dog는

학생회 회계 프로그램이 아니다.

AI가 회계를 관리해주는

Premium SaaS이다.

모든 화면은

디자인

사용성

애니메이션

타이포그래피

여백

컴포넌트

모두 최고 수준을 목표로 한다.

기능보다

경험(UX)을 우선한다.

디자인은

"예쁜 화면"

이 아니라

"신뢰를 만드는 경험"

을 제공해야 한다.
